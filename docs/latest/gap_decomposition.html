1. 每日收益差距分解 (shares=1)

柱体高度 = 当日 offline_gross − online_real（已除以 当日 segment shares：seg0=5, seg1–11=10, seg12–13=50，10→50 切换在 2026-05-11 13:58:19 UTC）。 六个组分堆叠：信号延迟(matched markets price2−price1)、 交易损失(matched price3−price2)、 data missing(bot API 漏掉最后 X 秒数据导致的错配)、 enter failure(only_offline，bot 也触发但未 fill)、 user_tag diff(换一个 tag 日期能让 SR 阈值跨越状态对上)、 其他(matched residual + 上面都解释不了的)。
注：offline_data_gap（only_online 市场，start_time 晚于 trades_alt parquet 最新数据 → Goldsky 还没 ingest 完）已从图表和详情区 完全过滤掉，因为这种延迟只是数据流的时间问题，不是策略/算法差异。 黑色 ◆ = 直接计算的当日总 gap (verification)。

1b. 每日数据缺失延迟 — average lag seconds for data_missing markets

对每个 data_missing 市场，复刻 offline SR 时丢掉最后 X∈{10, 20, 30, 40, 50, 60}s 的数据，找最小 X 让阈值跨越状态与 online 一致 → lag = X 秒。下图：每天的 mean lag + 90% 经验 CI (5/95 分位数) band。 点上 n=K = 当天 data_missing 市场数。

1c. 按 bot ladder shares 分组的成本分解

bot 在窗口里跑过几个不同 shares 配置（startup snapshot 里的 shares 字段：5/10/50）。每个 shares 区间画一张子图。 x 轴 = 6 个 cost 类别；y 轴 = 每个 market 在该类别上的 $/share 贡献， 画 bar = mean、errorbar = ±1.96·SE (95% CI of mean)。每个 bar 上写出 mean 值和样本数 n。每个子图标题写出该 shares 区间的 actual gap (offline − online，scale 到 1 share)。
